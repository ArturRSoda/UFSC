Revisão da escola - MDC
Deﬁnição: um número d é divisor de a se a = d * n, onde a, d, n são inteiros.
Exemplo: os divisores positivos de 24 são 1, 2, 3, 4, 6, 8, 12, e 24.
Deﬁnição: o máximo divisor comum entre a e b é o maior inteiro que divide
simultaneamente a e b.
Exemplo:
●
●

360 = 2³ * 3² * 5 e 84 = 2² * 3 * 7
mdc(360, 84) = 12 = 2² * 3

Revisão de mtm discreta - MDC
●

O cálculo do mdc é importante para diversos algoritmos criptográﬁcos
○

●
●

Se a e b são “relativamente primos”, isso signiﬁca que mdc(a,b) = 1

A técnica de fatoração para encontrar o mdc só funciona para números
compostos, cuja fatoração contém primos conhecidos/pequenos.
Como calculamos mdc de números grandes?
○
○
○

Algoritmo de Euclides
mdc(a, b) = mdc(b, a mod b)
Exemplo: mdc(360, 84) = mdc(84, 24) = mdc(24, 12) = mdc(12, 0) = 12

Algoritmo de Euclides
Algoritmo de Euclides:
sejam a = 360 e b = 84:
a=q×b+r
360 = (4) × 84 + 24
84 = (3) × 24 + 12
24 = (2) × 12 + 0

mdc(360, 84)

and network security. Cap 2.2

Mod e congruência
●

a mod n é o inteiro r tal que a = qn+r,
○
○
○
○
○

●

exemplo: 11 mod 7 = 4 pois 11 = 1 x 7 + 4
a mod n é o resto da divisão de a por n
n é chamado de módulo, r de resto
mod n mapeia qualquer número inteiro a para um número do conjunto {0, 1, …, n-1}
Obs.: se a mod n = 0, então a é divisível por n

a ≡ b (mod n) se a mod n = b mod n
○
○

0 <= r < n

24 ≡ 9 (mod 5) (r = 4)
Obs.: se a ≡ b (mod n), então b ≡ a (mod n)

●

Algoritmo de Euclides Estendido
●

Se mdc(a,n) = 1, sabemos que a-1 existe em ℤn
○
○

●
●

Para encontrá-lo, utilizamos o algoritmo de euclides estendido
Exemplo: seja a = 5 e n = 8, queremos encontrar a inversa de 5 em ℤ8

n=q⨉a+ r

5=1⨉3+2
3=1⨉2+1
2=2⨉1+0

mdc(5,8) = 1 = 3 - 1 ⨉ 2
= 3 - 1 ⨉ [5 - 1 ⨉ 3]
(substitui 2)
=3-1⨉5+1⨉3
(distributiva)
=2⨉3-1⨉5
( juntar)
= 2 ⨉ [8 - 1 ⨉ 5] - 1 ⨉ 5 (substitui 3)
= 2 ⨉ 8 - 2 ⨉ 5 - 1 ⨉ 5 (distributiva)
=2⨉8-3x5
( juntar)

inversa de 5 é -3 mod 8 = 5

(Pequeno) Teorema de Fermat
Teorema de Fermat: Se p é primo e a é um inteiro positivo não divisível por p
então
ap-1 ≡ 1 (mod p)
●

Requer que a e p sejam relativamente primos
○

●
●

mdc(a,p) = 1

Forma alternativa: p é primo e a é um inteiro positivo, então ap ≡ a (mod p)
Exemplo: p = 5, a = 3
○
○

○
○

ap-1 = 34 = 81 ≡ 1 (mod 5)
ap = 35 = 243 ≡ 3 (mod 5) = a (mod p)
3⁴⁰² (mod 5)
(34)100 x 3² ≡ 1100 x 3² ≡ 9 ≡ 4 (mod 5)

Teorema de Euler
Teorema de Euler: se a e n são relativamente primos, então
aϕ(n) ≡ 1 (mod n)
●
●
●

Note que esse teorema não assume n primo!
Forma alternativa: aϕ(n)+1 ≡ a (mod n)
Exemplo: a = 3, n = 10, mdc(3,10) = 1, ϕ(10) =ϕ(2)ϕ(5) = 1x4 = 4
○
○

aϕ(n) = 34 = 81 ≡ 1 (mod 10)
Versão alternativa: 35 = 243 ≡ 3 (mod 10)

●

Teorema de Fermat: Se p é primo e a é um inteiro positivo não divisível por p

○
○
○

esse teste é condição necessária mas não suﬁciente para garantirmos que n é primo

Começamos com algumas propriedades:

●

Se ﬁzermos esse teste com um número n e:
○

●

Para qualquer número ímpar n ≥ 3, podemos
escrever n - 1 = 2k m, onde k > 0 e m é a parte ímpar
do número.

○

Note que, se n é ímpar, então n-1 é par, estamos
dividindo n-1 por 2 k vezes, até chegar em m.

○

○

●
●

Seja p um primo maior que 2, podemos escrever
p-1 = 2k m.
Seja a qualquer inteiro 1 < a < p-1, uma das duas
condições é verdadeira (ver prova no Stallings):
○
○

am ≡ 1 (mod p) (ou seja, am mod p = 1); ou
um dos valores am, a2m, a4m, …, a(2^(k-1))m será congruente
à -1 módulo p
■
ou seja, a(2^i)m ≡ -1 (mod p), para algum 0 < i < k
■
Lembrete: -1 mod p = p-1

nenhuma dessas condições for satisfeita, n é
com certeza composto;
se alguma das condições for satisfeita, n não
necessariamente é primo.
Exemplo: n = 2047, n-1 = 2x1023,

composto.

Logaritmo discreto
●

●

Para qualquer inteiro b e raiz primitiva a de um
número primo n, podemos encontrar um único
expoente x tal que b = ax mod n
○ esse expoente é o logaritmo discreto do
número b para a base a (mod n)
○ representamos esse valor como dloga,n(b)
Exemplo:
○
○

dlog2,19(4) = 2 pois 4 = 22 mod 19
dlog2,19(18) = 9 pois 18 = 29 mod 19

and network security. Cap 2.8

Problema do logaritmo discreto
●

Considere a equação: b = ax mod p
○

●

Problema: dado b, a e p, geralmente é muito difícil encontrar x
○

●

dado a, x e p, é fácil e eﬁciente calcular b, basta fazer exponenciação
Acredita-se que a diﬁculdade é da mesma ordem de magnitude da diﬁculdade de fatorar os
números requeridos para o RSA

Força bruta: elevar a base a à diversas potências x até achar o valor b
○

Como 1 ≤ x ≤ p-1, se p for grande o suﬁciente, força bruta é computacionalmente inviável
